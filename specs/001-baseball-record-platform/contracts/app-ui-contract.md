# APP UI Contract

## 目的

APP 端聚焦於行動情境下的個人賽程、首頁摘要、賽前確認、比賽記錄、換人操作與分享預覽。APP 與 Web 共用同一套 mock 狀態與規則，但仍需遵守相同的個人/球隊/身分上下文邏輯。

## 共通約束

| 項目 | 說明 |
|------|------|
| 情境切換 | 與 Web 端共用 scenario 與 local state。 |
| 導覽方式 | `home.html` 為主要入口，其他頁面以卡片、按鈕與摘要區塊導覽。 |
| 資料來源 | 一律使用 shared core 與 `assets/data/*.json`。 |
| 展示原則 | 優先呈現可操作流程與狀態切換，不串接真實裝置能力。 |
| 上下文一致性 | APP 首頁需顯示目前使用者、當前球隊與當前身分摘要，並和 Web 共用同一套上下文。 |
| 未選球隊限制 | 未選擇球隊時以個人內容為主，不直接顯示球隊管理或球隊統計內容。 |

## 登入與首頁摘要

| 項目 | 說明 |
|------|------|
| 對應頁面 | `login.html`, `home.html` |
| 核心功能 | 顯示目前使用者、個人賽程摘要、個人成績入口、球隊摘要與主要功能入口。 |
| 必要狀態 | 未登入、已登入個人視角、多球隊、多身分、受限角色、待審核紀錄員情境。 |
| 互動要求 | 登入後首頁優先呈現個人資訊；若已選定球隊，可額外顯示該隊相關摘要。 |

## 賽程、通知與回覆

| 項目 | 說明 |
|------|------|
| 對應頁面 | `schedule.html`, `home.html` |
| 核心功能 | 顯示個人賽程、個人回覆、通知摘要與所屬球隊賽事分類。 |
| 必要狀態 | 參加、不參加、待定、未回覆、已讀/未讀通知。 |
| 互動要求 | 可更新個人回覆並看到首頁摘要同步變更；若切換球隊，需同步刷新該球隊相關賽程摘要。 |

## 賽前摘要與名單檢視

| 項目 | 說明 |
|------|------|
| 對應頁面 | `game-center.html`, `lineup.html` |
| 核心功能 | 顯示比賽摘要、規則、先發名單、替補、個人角色與紀錄權限狀態。 |
| 必要狀態 | 已確認名單、受限功能、臨時紀錄員申請中/已核准/已拒絕。 |
| 互動要求 | 需能看到規則摘要、球員可用性與目前紀錄權限；未選球隊時不得直接進入有效名單內容。 |

## 即時記錄與計分板

| 項目 | 說明 |
|------|------|
| 對應頁面 | `event-entry.html`, `scoreboard.html` |
| 核心功能 | 顯示逐筆事件輸入、待確認事件、時間線與計分板。 |
| 必要狀態 | 手動、掃描、語音 `captureSource`，以及 `pending` / `confirmed` 事件。 |
| 互動要求 | 掃描/語音事件需先進入待確認，再寫入正式時間線與計分板。 |

## 換人與再上場

| 項目 | 說明 |
|------|------|
| 對應頁面 | `substitutions.html` |
| 核心功能 | 顯示代打、代跑、守備更換、再上場限制與合法性結果。 |
| 必要狀態 | 合法換人、非法換人、友誼賽動態人數。 |
| 互動要求 | 變更後需立即更新合法性提示與場上狀態摘要。 |

## 分享預覽

| 項目 | 說明 |
|------|------|
| 對應頁面 | `share-preview.html` |
| 核心功能 | 依 A / B / C 分享層級預覽使用者可見內容。 |
| 必要狀態 | Tier A、Tier B、Tier C、未授權封鎖。 |
| 互動要求 | 切換層級後需同步更新卡片、摘要與封鎖提示。 |
