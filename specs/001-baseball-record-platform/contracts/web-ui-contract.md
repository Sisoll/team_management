# Web UI Contract

## 目的

Web 端必須呈現接近真實產品的使用流程：使用者進站後先看到首頁與登入入口，登入後預設進入行事曆首頁，再透過右上角全域操作區切換球隊與身分。所有頁面皆以靜態 HTML + JavaScript 運作，使用共用 mock 資料與 shared core 邏輯，不串接真實後端。

## 共通約束

| 項目 | 說明 |
|------|------|
| 情境切換 | 所有 Web 頁面需可讀取共用 scenario，支援 `baseline`、`new-user`、`restricted`、`scorer-request`。 |
| 導覽一致性 | Header / 右上角全域操作區 / 上下文顯示由共用 `common` 模組提供。 |
| 資料來源 | 一律來自 `assets/data/*.json`、`mock-store.js`、`permission-engine.js`、`stats-engine.js`，不得依賴即時 API。 |
| 顯示語言 | 介面以繁體中文為主。 |
| 體驗優先級 | 優先展示真實使用流程與狀態切換，而不是模組清單。 |
| 上下文一致性 | 所有頁面都必須顯示目前登入狀態、當前球隊、當前身分，且切換後同步反映。 |
| 未選球隊限制 | 未選擇球隊時只顯示個人相關內容；球隊成績、球隊管理、球隊名單與球隊賽程不得直接顯示。 |

## 首頁與全域操作區

| 項目 | 說明 |
|------|------|
| 對應頁面 | `index.html`, `auth.html`, `calendar.html` |
| 核心功能 | 顯示未登入首頁、登入入口、登入後的行事曆首頁，以及右上角登入/帳號/管理/球隊切換/身分切換。 |
| 必要狀態 | 未登入、已登入個人視角、已選球隊、未選球隊、多球隊、多身分、受限角色。 |
| 互動要求 | 登入後預設導向 `calendar.html`；右上角切換球隊或身分後，首頁、統計、隊務入口需同步更新。 |

## 個人視角與個人成績

| 項目 | 說明 |
|------|------|
| 對應頁面 | `calendar.html`, `stats.html` |
| 核心功能 | 在未選球隊時顯示個人賽程、個人通知、個人回覆與個人成績。 |
| 必要狀態 | 無球隊選取、球員個人成績、一般成員個人摘要、無統計資料、受限角色。 |
| 互動要求 | 不需先選球隊即可查看自己的個人成績；但不得混入任何未選定球隊的球隊層級統計。 |

## 球隊治理與管理入口

| 項目 | 說明 |
|------|------|
| 對應頁面 | `teams.html` |
| 核心功能 | 顯示球隊建立、成員邀請、角色矩陣、管理入口與球隊設定。 |
| 必要狀態 | 尚無球隊、已有多球隊、邀請中、管理者、教練、一般成員。 |
| 互動要求 | 建立球隊可在無球隊狀態下操作；既有球隊的管理內容必須在選定球隊後才顯示。 |

## 球員資料與歷史

| 項目 | 說明 |
|------|------|
| 對應頁面 | `players.html` |
| 核心功能 | 顯示球員名單、歷史節點、可用狀態、帳號連結與篩選。 |
| 必要狀態 | 未綁定帳號、後續補綁、離隊、回隊、不可出賽、空名單。 |
| 互動要求 | 只有在當前球隊已選定時才顯示有效名單；切換球隊後名單內容需同步切換。 |

## 比賽建立與先發規劃

| 項目 | 說明 |
|------|------|
| 對應頁面 | `games.html`, `lineup.html` |
| 核心功能 | 顯示比賽建立、規則模板切換、先發打序、守位與替補規劃。 |
| 必要狀態 | 棒球/壘球、正式賽/友誼賽、DH/EP、再上場限制、名單合法/不合法。 |
| 互動要求 | 未選球隊時不得直接進入有效配置；切換規則模板後需立即刷新合法性與摘要。 |

## 行事曆、通知與回覆

| 項目 | 說明 |
|------|------|
| 對應頁面 | `calendar.html` |
| 核心功能 | 以同一頁面承接個人賽程與球隊賽程；未選球隊時預設顯示個人賽程，選球隊後顯示該隊賽程與回覆統計。 |
| 必要狀態 | 個人視角、球隊視角、參加、不參加、待定、未回覆、截止時間、通知已讀/未讀。 |
| 互動要求 | 個人回覆變更後需同步更新個人摘要；球隊管理者在選定球隊後可查看該隊回覆統計。 |

## 賽事回顧與統計

| 項目 | 說明 |
|------|------|
| 對應頁面 | `review.html`, `stats.html` |
| 核心功能 | 顯示事件回放、修正、個人統計、球隊統計與視角切換。 |
| 必要狀態 | 原始事件、修正後事件、個人視角、球隊視角、特殊結果摘要。 |
| 互動要求 | 修正事件後需同步刷新 timeline、個人成績與球隊統計；個人視角只顯示自己的資料。 |

## 戰績報表

| 項目 | 說明 |
|------|------|
| 對應頁面 | `reports.html` |
| 核心功能 | 顯示分類戰績、3 個月 / 1 年 / 自訂區間報表與合併模式切換。 |
| 必要狀態 | 正式賽與友誼賽預設分開、延賽/棄賽/判定勝負等特殊結果。 |
| 互動要求 | 僅在當前球隊已選定時顯示有效內容；預設不將特殊結果納入戰績，可切換合併檢視模式。 |

## 分享層級

| 項目 | 說明 |
|------|------|
| 對應頁面 | `share.html` |
| 核心功能 | 顯示 A / B / C 公開層級、授權可見欄位與封鎖訊息。 |
| 必要狀態 | Tier A、Tier B、Tier C、未授權檢視。 |
| 互動要求 | 切換分享層級時內容需依權限過濾；未授權時顯示封鎖提示。 |
